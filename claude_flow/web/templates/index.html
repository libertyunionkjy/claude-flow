Claude Flow Workflow Guide

Claude Flow uses a task queue + Git Worktree isolation model for multi-instance parallel development.
Below is the complete lifecycle of a task -- click each step to expand details.

1 Initialize Project Setup ▼

First, initialize Claude Flow in your Git project root. This creates a .claude-flow/ directory containing configuration and task data.

CLIcf init
WebAuto-initialized when Web Manager starts
Tip: Run cf init once per project. It creates tasks.json, config.json, and a worktrees/ directory.
2 Add Tasks Pending ▼

Create one or more tasks with a title and a prompt. The prompt is what Claude Code will receive as instructions. Tasks start in Pending status.

CLIcf task add -p "prompt" "title"
WebClick + New Task button in the toolbar
Tip: Use -f tasks.txt to batch-add from a file, one task per line. Set priority with the -P flag (0-3).
3 Generate Plans Planning AUTO ▼

Choose a planning mode for each task:

  • Auto Plan - AI generates a complete plan directly (best for bug fixes and small features)
  • Chat Plan - Multi-round conversation to refine requirements and design before generating the plan
CLIcf plan <task_id> (auto) or cf plan <task_id> --interactive
WebClick Auto Plan or Chat Plan on the card

Both modes produce a Markdown plan document in .claude-flow/plans/.

4 Review & Approve Plans Planned HUMAN ▼

This is the human-in-the-loop gate. Review each plan to ensure correctness and safety before approving execution.

CLIcf plan review
cf plan approve --all
WebClick View Plan to read, then Approve or Chat
Chat: Open the chat dialog to discuss the plan with AI, ask questions, or request revisions. Click Generate Plan in the chat to regenerate from the conversation.
5 Execute Tasks Approved AUTO ▼

Workers pick up approved tasks and execute them. Each task gets its own Git Worktree (isolated branch cf/task-XXXXXX) so multiple tasks can run in parallel without conflicts.

CLIcf run  or  cf run -n 3
WebClick Run All or card's Run button

The worker calls claude -p "prompt" inside the worktree. The task moves to Running.

6 Running & Needs Input Running ▼

While running, Claude Code writes code in the isolated worktree. Monitor progress via logs.

If Claude Code needs clarification, the task enters Needs Input status. You can provide additional context and the task will resume.

CLIcf log <task_id>
cf status
WebClick View Log on running cards, or Respond for Needs Input cards
7 Merge & Done Done AUTO ▼

After execution completes, the worker automatically merges the worktree branch back to the main branch (configurable via auto_merge). The worktree is then cleaned up.

CLIcf clean
WebTask card moves to Done column automatically
Config: Set auto_merge: false in config.json to merge manually. Merge strategy defaults to --no-ff.
8 Handle Failures Failed HUMAN ▼

If a task fails (timeout, Claude error, merge conflict), it enters Failed status with an error message. You can:

  • Retry -- re-queue the task as Approved
  • Reset -- send it back to Pending (start from scratch)
  • View Log -- inspect what went wrong
  • Delete -- remove the task entirely
CLIcf retry
cf reset <task_id>
WebUse Retry, Reset, or Delete buttons on failed cards
